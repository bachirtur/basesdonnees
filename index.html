1 Introduction aux Bases de Données

🤔 Qu'est-ce qu'une base de données ?

Une base de données est un système organisé permettant de stocker, gérer et retrouver des informations de manière structurée. Pensez-y comme une armoire bien rangée où chaque tiroir contient des informations classées.

🔗 Qu'est-ce qu'une base de données relationnelle ?

Il existe différents types de bases de données (hiérarchiques, objet, NoSQL…). Les bases de données relationnelles sont les plus utilisées au monde.

Elles ont été mises au point en 1970 par Edgar Frank Codd, informaticien britannique (1923-2003). Elles sont basées sur la théorie mathématique des ensembles.

La notion de relation (ou table) est au cœur de ce type de base de données. Une relation peut être vue comme un tableau à 2 dimensions.

📖 Vocabulaire essentiel

📋
Table (Relation)
Un tableau à 2 dimensions composé d'un en-tête et d'un corps. C'est la structure de base pour organiser les données.
↔️
Ligne (T-uplet)
Une ligne du tableau. Chaque t-uplet représente un enregistrement, par exemple un livre précis avec toutes ses informations.
↕️
Colonne (Attribut)
Une colonne du tableau. Chaque attribut représente une propriété, par exemple le titre, l'auteur ou la note d'un livre.
🔢
Domaine
L'ensemble des valeurs admissibles pour un attribut. Par exemple : INT (entiers), TEXT (chaînes de caractères).

⚠️ Problème : les doublons

✅ Doublons supprimés ! Chaque t-uplet est maintenant unique grâce à la clé primaire (l'attribut id).

🔑 Qu'est-ce qu'une clé primaire ?

Une clé primaire est un attribut dont la valeur permet d'identifier de manière unique chaque t-uplet de la relation. On ne doit jamais trouver deux fois la même valeur pour cet attribut.

Peut-on utiliser ces attributs comme clé primaire ?

📝 Notation du schéma relationnel

LIVRES(id : INT, titre : TEXT, id_auteur : INT, ann_publi : INT, note : INT)

Les attributs soulignés sont des clés primaires.

4 Clé Étrangère et Relations

🔗 Pourquoi utiliser plusieurs tables ?

Si on met toutes les informations dans une seule table, on obtient de la redondance : les mêmes données sont répétées inutilement. Par exemple, les informations d'un auteur seraient répétées pour chacun de ses livres.

La solution : créer deux tables séparées et établir un lien entre elles grâce à une clé étrangère.

📊 Les deux tables liées

Survolez un livre pour voir le lien vers son auteur. Cliquez sur une valeur de id_auteur pour mettre en évidence la correspondance.

📖 Relation LIVRES

id_auteur → id

✍️ Relation AUTEURS

📝 Définitions

Clé étrangère : Un attribut d'une relation RA qui prend les valeurs de la clé primaire d'une autre relation RB. Cela permet d'établir un lien entre les deux relations.

Schéma relationnel complet

AUTEURS(id : INT, nom : TEXT, prenom : TEXT, ann_naissance : INT, langue_ecriture : TEXT)

LIVRES(id : INT, titre : TEXT, #id_auteur : INT, ann_publi : INT, note : INT)

Les attributs soulignés sont des clés primaires. Le # indique une clé étrangère.

5 Console SQL Interactive

💻 Qu'est-ce que SQL ?

SQL (Structured Query Language) est le langage utilisé pour interroger et manipuler les bases de données relationnelles. Voici les commandes de base :

SELECT * FROM livres → Sélectionner toutes les colonnes de la table livres
SELECT titre FROM livres → Sélectionner uniquement la colonne titre
SELECT * FROM livres WHERE note > 8 → Filtrer les livres ayant une note supérieure à 8
SELECT titre, nom FROM livres JOIN auteurs ON livres.id_auteur = auteurs.id → Joindre les tables livres et auteurs

6 Exercices de Pratique

🎯 À vous de jouer !

Pour chaque exercice, écrivez la requête SQL dans la console puis vérifiez avec la solution.

Exercice 1 Facile

Affichez tous les livres avec toutes leurs informations.

SELECT * FROM livres
Exercice 2 Facile

Affichez uniquement les titres de tous les livres.

SELECT titre FROM livres
Exercice 3 Moyen

Trouvez tous les livres publiés après 1950.

SELECT * FROM livres WHERE ann_publi > 1950
Exercice 4 Moyen

Affichez tous les auteurs avec toutes leurs informations.

SELECT * FROM auteurs
Exercice 5 Moyen

Trouvez les livres qui ont une note supérieure ou égale à 9.

SELECT * FROM livres WHERE note >= 9
Exercice 6 Avancé

Affichez le titre de chaque livre avec le nom et le prénom de son auteur (jointure entre les deux tables).

SELECT titre, nom, prenom FROM livres JOIN auteurs ON livres.id_auteur = auteurs.id
Exercice 7 Avancé

Affichez les auteurs qui écrivent en français.

SELECT * FROM auteurs WHERE langue_ecriture = 'français'
